## Fix 2 — Edge label clipping

Two edge labels are clipped. Investigate why:

```
grep -n "per-BU\|preference" src/pulsecraft/demo/static/architecture.js
```

The labels are positioned along arrows between nodes. If the arrow is short (because adjacent nodes are close together), the label text extends past the arrow ends and gets clipped by the node boxes.

**Approach A — shorten the labels:**
- "per-BU brief" → "brief" (the per-BU nature is already communicated by BUAtlas's stacked-card visual and sub-label)
- "preference" → "preference" → "decision" or "pref"

This is simplest. The words are aesthetic labels, not critical information.

**Approach B — reposition labels to sit above or below the arrow, not on it:**

If labels are positioned via `text-anchor: middle` on the arrow midpoint, they may overlap node edges. Move the `<text>` element's `y` attribute up or down by ~14px so the label sits clear of the node boxes.

**Approach C — ensure nodes have more horizontal gap** between them so labels fit. Requires recalculating node x-positions.

**Recommendation:** Combine A + B. Shorten "per-BU brief" to "brief" and "preference" to "pref" or "decision" (Approach A), AND move both labels slightly above the arrow line so they float clear of node tops/bottoms (Approach B).

**Verify:** after the fix:
- The label previously showing "er-BU brief" now reads fully as "brief" (or the shortened version you chose)
- The label previously showing "eferen" now reads fully as "pref" or "decision"
- Neither label is clipped by adjacent node boxes
