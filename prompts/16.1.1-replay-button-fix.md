# Prompt 16.1.1 — Replay Button Overlap Fix

> **Character.** One surgical fix: the replay button overlaps the `pre_deliver` node because it's absolutely positioned inside the canvas. Move it out.
>
> **Expected duration:** 15 minutes.
>
> **Budget:** Zero LLM cost.

---

## Problem

`_buildReplayBtn()` appends the button to `#arch-canvas` as `position: absolute; top: 16px; right: 16px`. The rightmost pipeline node (`pre_deliver`) sits at `cx=1000` in a `SVG_W=1100` canvas, so the button visually overlaps it at any common viewport width.

## Fix

Move the replay button out of `#arch-canvas` into `#arch-heading` as a flex sibling of the heading text. The button lives at a structurally different level — it can never overlap pipeline content regardless of viewport width.

### `architecture.js` changes

1. `_buildDOM()`: give the heading div `id="arch-heading"` and wrap the h2+p in a `<div class="arch-heading__text">` container.
2. `_buildReplayBtn()`: target `document.getElementById('arch-heading')` instead of `document.getElementById('arch-canvas')`.

### `architecture.css` changes

1. `.arch-heading` → `display: flex; align-items: flex-start; justify-content: space-between; gap: 24px;`
2. Add `.arch-heading__text { flex: 1 1 auto; min-width: 0; }`
3. `.arch-canvas` → remove `position: relative` (no longer an anchor for the button)
4. `.arch-replay-btn` → remove `position: absolute; top: 16px; right: 16px; z-index: 1; box-shadow`; add `flex: 0 0 auto; align-self: flex-start`
